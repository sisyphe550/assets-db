## 4. 发现并修复的 Bug

### 4.1 bcrypt hash 不匹配（严重）

**现象**：所有用户登录返回 40101，即使密码正确。

**原因**：`05-seed-fixtures.md` 中的 bcrypt hash 与密码 `Test@123456` 不匹配（可能来自不同的 bcrypt 实现或 cost 参数）。

**修复**：用 `bcrypt.GenerateFromPassword([]byte("Test@123456"), bcrypt.DefaultCost)` 重新生成 hash `$2a$10$BuwmRTR/mblICvn4jrLA0.9LVPRTEMdlQZBa7rMnttf6mieWX9pre`，同步更新 `002_seed.sql` 和 `05-seed-fixtures.md`。

### 4.2 组织树 rootID 错误（中等）

**现象**：role=2 院级管理员调用 GET /departments/tree 返回空列表。

**原因**：`DeptTree` handler 中 `dept.ToTree(filtered, 0)` 始终以 parent_id=0 为根，但院级管理员的子树中不存在 parent_id=0 的节点。院级管理员需要以其自身部门为根。

**修复**：role=1 使用 rootID=0；role=2 以管理员 deptID 为根，手动构造根节点再挂载 children。

### 4.3 校级资产列表 role=1 返回空（中等）

**现象**：校级管理员 GET /assets 返回空列表。

**原因**：`List` handler 依赖 `GetDeptSubtree` 中间件注入的 deptIDs，但 main.go 只注册了 `JWTAuth` 而未注册 `RequireDeptScope`，导致 `unlimited=false` + `deptIDs=[]`。

**修复**：在 `List` handler 中增加 role=1 的直接判断，role=1 视为 unlimited。

> **2026-07-09 补充**：院级管理员（role=2）资产列表为空是另一问题——asset-api 未展开院系子树。修复见 `13-release-notes-2026-07-09.md` §1.1（`college-subtree` fallback + 详情 scope 校验）。

### 4.4 Workflow struct 缺少 json tag（轻微）

**现象**：工作流 API 返回 JSON 字段名为 PascalCase（如 `AssetID` 而非 `assetId`），导致前端/Python 解析失败。

**原因**：`WorkflowRequest` 和 `WorkflowLog` 只定义了 `db` tag 未定义 `json` tag。

**修复**：添加 `json:"camelCase"` tag。

### 4.5 inventory_task 表缺少 updated_at 列（轻微）

**现象**：POST /tasks/:id/archive 返回 50001。

**原因**：DDL 中 `inventory_task` 表无 `updated_at` 列，但 `ArchiveTask` 函数引用了它。

**修复**：从 UPDATE 语句中移除 `updated_at=NOW()`。

### 4.6 所有 PG 表 id 列无序列（严重）

**现象**：INSERT 操作失败，因为 id 是 NOT NULL 但没有 AUTO_INCREMENT/SEQUENCE。

**原因**：DDL 设计为显式 ID 管理但实际代码依赖自动生成 ID。

**修复**：所有表执行 `ALTER TABLE ... ALTER COLUMN id ADD GENERATED BY DEFAULT AS IDENTITY`。同时 `CREATE DATABASE fams_report`。
